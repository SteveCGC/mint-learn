## 1. 全局功能

全局功能贯穿所有页面，不属于单一页面的功能。

### 1.1 顶部导航栏（Navbar）

| 元素 | 功能 |
|------|------|
| Logo / 站名 | 点击跳转首页 |
| 导航链接 | 首页 · 课程广场 · 发布课程（登录后显示）|
| 语言切换 | 中文 / English 切换（URL 路由级别，`/zh/` ↔ `/en/`）|
| 钱包连接按钮 | 未连接：「连接钱包」；已连接：显示地址缩写 + 网络徽标 |
| 个人头像/菜单 | 登录后显示，下拉包含：个人中心 · 收益中心 · 退出登录 |
| 网络警告 | 若当前网络非 Sepolia，顶部显示橙色提示条 + 「切换网络」按钮 |

### 1.2 钱包连接流程（全局 Modal）

```
点击「连接钱包」
  │
  ├── 未安装 MetaMask → 弹窗提示「请安装 MetaMask」，附安装链接
  │
  └── 已安装 → MetaMask 弹出授权 → 获取 address
                │
                └── 检测网络 → 非 Sepolia → 弹窗引导「切换到 Sepolia 测试网」
```

### 1.3 签名登录流程（全局）

钱包连接成功后，访问需要登录的页面时自动触发：

```
GET /auth/nonce（获取随机 nonce）
  │
  └── MetaMask 弹出签名弹窗（说明：非交易，不消耗 Gas）
        │
        ├── 用户拒绝 → Toast 提示「登录已取消」
        │
        └── 用户确认 → POST /auth/verify → 签发 JWT Cookie
                          │
                          └── 成功 → 刷新登录态，继续原操作
```

### 1.4 全局交易状态组件（TxProgress）

所有链上交易共用，底部 Sheet 浮层形式：

| 状态 | 展示内容 |
|------|---------|
| 等待确认 | 转圈动画 + 「请在 MetaMask 中确认」|
| 已广播 | 进度条 + txHash 链接（Sepolia Etherscan）|
| 已确认 | 绿色✓ + 「交易成功」+ 可选跳转 |
| 失败 | 红色✕ + 失败原因 + 「重新发送」按钮 |

### 1.5 底部 Footer

- 平台介绍一句话
- GitHub 链接
- 语言切换（同导航栏）
- 当前网络标识（Sepolia Testnet）

### 4.2 购买区（右侧卡片）

根据用户状态展示不同内容：

**状态一：未连接钱包**
```
[连接钱包购买]
```

**状态二：已连接，MT 余额不足**
```
MT 余额：X MT（不足）
[如何获取 MT 代币？] ← 跳转说明页或 Faucet
```

**状态三：已连接，余额充足，未授权**
```
MT 余额：X MT ✓
第一步：授权 MT 代币
[授权 CourseManager 合约]
→ MetaMask 弹出 approve 确认
```

**状态四：已授权，未购买**
```
MT 余额：X MT ✓
授权额度：✓
[立刻购买] ← 触发 purchaseCourse 链上交易
```

**状态五：已购买**
```
✓ 已购买
[进入课程] ← 跳转 /profile/[courseId]
```

### 4.3 购买流程（Step 组件）

```
Step 1  检查 MT 余额
Step 2  ERC20 approve（弹出 MetaMask）
Step 3  purchaseCourse 交易（弹出 MetaMask）
Step 4  等待链上确认（进度条）
Step 5  解锁成功 → 显示「进入课程」按钮
```

## 5. 发布课程

**路由：** `/[locale]/publish`
**访问权限：** 需登录（钱包连接 + JWT）

### 5.1 基础信息表单

| 字段 | 类型 | 验证 |
|------|------|------|
| 课程标题 | 文本 | 必填，1–100 字 |
| 课程简介 | Markdown 编辑器 | 选填，最多 2000 字 |
| 封面图片 | 图片上传 | 必填，≤ 5MB，JPG/PNG/WebP |
| 课程内容文件 | 文件上传 | 必填，≤ 500MB，支持 MP4/PDF/ZIP |
| 课程价格 | 数字输入 | 必填，> 0 MT，整数 |

### 5.2 发布流程

```
① 用户填写表单，点击「发布」
       │
② 前端上传封面图到 R2（POST /content/upload-url → 获取预签名上传 URL）
       │
③ 前端上传课程内容文件到 R2
       │
④ 前端调用 CourseManager.createCourse(price, metaHash)
   → MetaMask 弹出确认
       │
⑤ 等待链上确认（1–2 个区块）
       │
⑥ 链上确认后，POST /courses 将元数据写入 Supabase
   （chainId = 链上返回的 courseId，contentKey = R2 Object Key）
       │
⑦ 发布成功 → Toast 提示 + 跳转至课程详情页
```

### 5.3 上传进度展示

- 封面图上传进度条
- 内容文件上传进度条（大文件）
- 链上交易状态（同 TxProgress 组件）

### 5.4 草稿功能（可选，后续迭代）

- 表单数据自动存储到 `localStorage`
- 页面刷新后恢复未发布草稿

## 7. 课程学习

**路由：** `/[locale]/profile/[courseId]`
**访问权限：** 需登录 + 已购买该课程

### 7.1 访问鉴权流程

```
进入页面
  │
  ├── 未登录 → 重定向到首页，Toast「请先登录」
  │
  └── 已登录 → POST /content/access { courseId }
                  │
                  ├── 未购买（403）→ 重定向到课程详情页，Toast「请先购买课程」
                  │
                  └── 已购买 → 返回预签名 URL（15 分钟有效）
                                │
                                └── 前端加载课程内容
```

### 7.2 内容播放器 / 阅读器

根据课程内容类型自动适配：

| 内容类型 | 展示方式 |
|---------|---------|
| MP4 视频 | HTML5 Video Player（支持进度、倍速、全屏）|
| PDF 文档 | PDF 内嵌阅读器（iframe 或 react-pdf）|
| ZIP 压缩包 | 下载按钮（预签名 URL 直接下载）|

### 7.3 侧边栏

| 功能 | 说明 |
|------|------|
| 课程标题 | |
| 作者信息 | 昵称 + 地址，可跳转作者主页 |
| 返回按钮 | 「← 我的课程」回到 `/profile` |
| URL 刷新 | 预签名 URL 14 分钟后自动静默刷新（防内容中断）|

### 7.4 签名过期处理

- URL 有效期 15 分钟，前端在第 14 分钟静默调用 `/content/access` 刷新 URL
- 若 JWT 已过期（401）→ 弹出签名弹窗重新登录，不中断当前播放

### 8.3 AAVE 质押操作

#### 质押（Supply）

```
① 选择质押资产：MT / ETH / USDT
② 若选 MT 或 ETH：展示「兑换为 USDT」步骤（接入 DEX，可选）
③ 输入质押数量（显示可用余额，「最大」快捷按钮）
④ 点击「一键质押」
    │
    ├── Step 1: approve USDT → AAVE Pool（MetaMask 确认）
    │
    └── Step 2: AAVE Pool.supply()（MetaMask 确认）
                │
                └── 等待链上确认 → 成功 → 更新余额显示
```

#### 取回（Withdraw）

```
① 输入取回数量（显示当前质押余额，「全部取回」快捷按钮）
② 点击「取回资产」→ MetaMask 确认 AAVE Pool.withdraw()
③ 等待链上确认 → 成功 → 更新余额显示
```

## 11. 全局状态与交互组件

### 11.1 组件清单

| 组件 | 文件路径 | 使用场景 |
|------|---------|---------|
| `ConnectButton` | `components/wallet/ConnectButton.tsx` | Navbar，所有页面 |
| `SignatureModal` | `components/wallet/SignatureModal.tsx` | 签名登录时自动触发 |
| `TxProgress` | `components/wallet/TxProgress.tsx` | 所有链上交易 |
| `NetworkGuard` | `components/wallet/NetworkGuard.tsx` | 非 Sepolia 时显示警告 |
| `CourseCard` | `components/course/CourseCard.tsx` | 首页精选、课程广场 |
| `PurchaseFlow` | `components/course/PurchaseFlow.tsx` | 课程详情购买区 |
| `CourseForm` | `components/course/CourseForm.tsx` | 发布课程 / 编辑课程 |
| `StakePanel` | `components/defi/StakePanel.tsx` | 收益中心质押操作 |
| `EarningsChart` | `components/defi/EarningsChart.tsx` | 收益中心趋势图 |

### 11.2 页面间数据流

```
wagmi（链上实时数据）
  ├── useAccount()              → 全局钱包状态
  ├── useReadContract()         → MT 余额、allowance、hasPurchased
  └── useWaitForTransactionReceipt() → 交易确认后刷新相关数据

TanStack Query（服务端数据）
  ├── GET /courses              → 课程列表缓存
  ├── GET /users/me             → 个人信息缓存
  ├── GET /users/me/courses     → 已购课程列表
  └── GET /aave/positions       → 质押记录
```

### 11.3 错误边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 钱包未安装 | Modal 提示安装 MetaMask |
| 网络错误（非 Sepolia） | 顶部 Banner + 引导切换 |
| 签名被拒绝 | Toast「登录已取消」，不阻断浏览 |
| 交易 Gas 不足 | Toast 提示 + 「重新发送」按钮 |
| 链上交易失败 | TxProgress 红色失败态 + 原因展示 |
| JWT 过期（401）| 自动触发重新签名弹窗 |
| R2 URL 过期 | 静默刷新（第 14 分钟）|
| AAVE 操作失败 | Toast「操作失败，资产已安全回滚」|
