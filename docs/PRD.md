
# PRD: MintLearn — 去中心化课程学习平台

### TL;DR

MintLearn 是一款部署在 Sepolia 测试网的去中心化课程平台，任何人都可以自由上传课程，学员通过购买 MT 代币参与学习，平台将课程收益利用 AAVE 进行 DeFi 理财增值。平台采用 MetaMask 签名机制，实现账户安全和课程内容安全获取，真正做到人人可参与、数据自掌控。

---

## 目标

### 业务目标

* 实现 MT 代币在平台内的持续健康流通与生态自运行。

* 增加课程交易量和课程总上架数，提升平台活跃度。

* 保障通过 AAVE 的 DeFi 理财功能实现收益增值，达到连续正回报。

* 构建透明的课程收益分发与可追溯机制。

* 促进更广泛的课程上传与生态内容繁荣。

### 用户目标

* 用户可低门槛自由发布任何课程，提升内容供给多样性。

* 学员安全、轻松地用 MT 代币购买课程，体验快捷支付。

* 通过个人中心可视化查看收益与已购买课程，增强信任与粘性。

* 课程作者可便捷地将收益一键质押到 AAVE，获得被动理财收益。

* 所有交互过程都基于链上签名，账号私密安全，课程内容有效保障.

### 非目标

* 暂不支持多链与非 EVM 生态，聚焦 Sepolia 及 EVM 测试网。

* 当前版本不部署到主网，仅供测试和迭代使用。

* 不开发原生移动端应用，聚焦 Web 端体验。

## 功能需求

* 合约层（高优先级）MT 代币合约 ERC20：原生平台代币发行与转账，符合 ERC20 标准结构。课程合约部署：支持课程元数据上链，记录作者、价格、购买状态等核心字段。AAVE 集成合约：支持将 MT、ETH 或 USDT 等资产存入 AAVE 协议进行质押和取回。


* 课程管理（高优先级）课程创建/上架：所有用户都可填写元数据（标题、简介、内容 IPFS 链接等）自主发布课程。设置课程价格：作者自主定价，以 MT 币为单位，价格写入合约。课程状态维护：支持课程上架、下架、编辑和归档操作。


* 购买流程（高优先级）购买 MT 代币：集成常规代币购买与充值流程，适配 MetaMask。ERC20 approve 授权：学员对课程合约进行代币支出授权。课程购买：支持链上扣款与课程购买交易，防止双花和合约复用漏洞。


* 个人中心（中优先级）MetaMask 签名登录：用户通过钱包签名方式登录，无需传统注册。已购课程列表：安全拉取链上数据，展示课程摘要。课程详情安全获取：通过签名验证授权后获取敏感课程内容（链下存储）。


* 理财模块（中优先级）作者收益一键质押 AAVE：将 MT、ETH 或兑换 USDT 资产进行 AAVE 质押操作。资金流动与理财表现可视化：展示质押余额、收益率等数据。

## 用户体验

**入口与首次体验**

* 用户访问 Web 端首页，页面引导连接 MetaMask 钱包。

* 首次用户完成钱包授权后，将看到简洁的新手引导，包括课程浏览、代币购买指南、平台机制说明。

* 注册与登录均通过 MetaMask 签名完成，无传统 Web2 账户体系。

**核心体验流程**

* 第一步： 学员进入平台，点击“代币购买”一键跳转至 MT 代币获取页面。UI 提醒当前钱包网络要求及余额校验。失败时提示原因（余额不足、连接错误等）。

* 第二步： 学员挑选心仪课程，点击“立刻购买”。弹窗提示需先进行 ERC20 approve 授权，手把手引导。授权成功后自动跳转购买流程。

* 第三步： 购买课程链上确认，过程中展示交易状态与进度条。支持“再次发送”或“取消”功能。

* 第四步： 购买成功后，通过 MetaMask 签名方式进入个人中心。系统拉取个人已购列表，点击课程详情时，进行一次快速签名授权，安全获取内容。支持用户在个人中心自主修改名称（签名授权提交）。

* 第五步： 课程作者后台，支持一键发布课程，填写内容与上链存证。收益到账后，一键将 MT 或 ETH 按当前额度兑换成 USDT 并质押 AAVE。AAVE 流程提供实时进度与收益可视化页面。

**高级功能与边界情况**

* 授权额度不足、链上交易失败，提供明确的 UI 提示与恢复引导。

* AAVE 质押失败则回滚操作，并告知用户失败详情，避免资产误损。

* 钱包签名如果过期，则要求用户重新发起签名，防止重放攻击。

* 支持断网、网络变更等临时异常自动恢复和提示。

**UI/UX 设计亮点**

* 钱包连接状态醒目展示，随时可切换/刷新。

* 重要交易步骤均有进度条、动画反馈及错误友好提示。

* 签名弹窗配简洁明了组件，帮助用户理解签名本质和后果。

* 整体界面响应式适配主流 Web 端设备，重视低延迟与高对比度配色，保障可访问性。

## 技术架构

### 技术需求

* 实现 ERC20（MT）代币合约及平台课程管理合约（支持创建、定价、购买功能）。

* 课程序列与关联收益的链上数据模型。

* 对接 AAVE 协议（使用 Sepolia 地址，参考官方地址），合约支持存入与查询资产逻辑。

* 前端通过 Web3.js/Ethers.js 调用合约及 MetaMask Provider API，实现所有链上与签名操作（前端专用）。

* 前端技术栈：Next.js（部署至 Cloudflare Pages，需配置 @cloudflare/next-on-pages 适配器）。CSS 使用 Tailwind CSS，组件库采用 shadcn/ui（基于 Radix UI，组件源码直接集成到项目中，完全可定制）。shadcn/ui 的 Dialog、Toast、Progress、Sheet 等组件用于钱包连接弹窗、交易进度条、签名确认弹窗等核心交互场景。前端通过 Ethers.js 调用合约及 MetaMask Provider API。

* 国际化支持：国际化框架采用 next-intl，并与 Next.js 深度集成，支持基于路由的语言切换（如 /en/courses、/zh/courses）。翻译文件统一存放在 packages/i18n 工作区，前后端共享，避免重复维护。初期支持中文（zh）和英文（en）两种语言，架构上预留多语言扩展能力，后续可低成本新增其他语言。日期、时间显示按用户浏览器时区自动处理。课程定价以 MT 代币计价，无货币本地化问题。错误提示、交易状态、签名弹窗等所有用户可见文案均纳入 i18n 翻译体系，不硬编码。


* Monorepo 与包管理：采用 pnpm workspaces 管理，推荐项目结构如下：

  * `apps/web`：Next.js 前端应用，部署到 Cloudflare Pages。

  * `apps/api`：Hono + Cloudflare Workers 后端 API。

  * `packages/contracts`：Solidity 智能合约。

  * `packages/prisma`：Prisma schema 与 migrations，供 api 共享。

  * `packages/types`：共享 TypeScript 类型定义，前后端复用。

  * `packages/i18n`：i18n 语言包，前后端共享。

  * 根目录 `package.json` 配置 pnpm workspace。

* 后端验证层采用 Cloudflare Workers 实现，后端 API 使用 Hono 框架运行在 Cloudflare Workers 上：Hono 负责所有路由，包括签名验证、课程管理 API 以及为 R2 对象生成受限的短时效预签名 URL。Workers 在验证用户提交的 MetaMask 签名并查询链上/数据库购买记录后，授予前端对课程内容的访问权限。Prisma 作为 ORM，用于类型安全地与 Supabase PostgreSQL 交互。Prisma 通过 Supabase 内置的 PgBouncer 连接池连接 PostgreSQL，DATABASE_URL 指向 Supabase PgBouncer 地址（示例包含参数 ?pgbouncer=true&connection_limit=1），directUrl 指向直连地址用于迁移。部署 Workers 时建议使用适配 Workers 的 Prisma 配置与命令（例如：prisma generate --no-engine）。课程内容（视频、文档等大文件）存储在 Cloudflare R2；Cloudflare Workers（通过 Hono 路由）负责生成受限的、短时效的预签名 URL 以供客户端下载。课程元数据仍保留上链或存储于 Supabase 以保证透明性与可查询性。

### 集成点

* AAVE V3 Sepolia 合约，所有理财功能依赖于 AAVE 接口的准确可用性。

* MetaMask 钱包 Provider，所有账号管理与签名交互依赖于其 API 兼容性（前端使用 Web3.js/Ethers.js）。

* Cloudflare Workers + Cloudflare R2：作为后端验证与内容分发层，负责签名验证、链上购买校验与对 R2 生成短时效预签名 URL 的能力。

* Cloudflare Pages：作为前端部署平台，和 Workers、R2 同处 Cloudflare 生态，允许统一域名管理并能享受零出口流量费用（减少跨服务带来的流量成本与运维复杂度）。

* Supabase PostgreSQL：Supabase 提供 PostgreSQL 数据库，内置 PgBouncer 连接池以解决 Workers 的 TCP 连接限制，同时提供 Row Level Security（RLS）作为额外的数据访问保护层，Dashboard 可视化管理与完善的生态工具链。相比于 Cloudflare Hyperdrive，Supabase 在关系型数据库与运维支持上更成熟稳定，适合作为平台的核心关系型数据存储与查询后端。

* Hono + Prisma：作为后端层部署在 Cloudflare Workers 之上，位于前端与区块链 / Supabase 之间，负责路由、签名验证、对 Supabase 的数据库查询（通过 Prisma）、以及生成 R2 presigned URL。

* packages/i18n 作为共享翻译包，供 apps/web 和 apps/api 共同引用，保证前后端错误信息、通知文案语言一致。

### 部署与 CI/CD

**部署策略：前后端分别独立部署**

* 前端（apps/web）部署至 Cloudflare Pages，后端（apps/api）部署至 Cloudflare Workers，两者独立部署、独立回滚，互不影响。

* 智能合约（packages/contracts）单独手动部署至 Sepolia 测试网，不纳入自动化流水线。

**GitHub Actions 自动化流水线**

* `deploy-web.yml`：监听 `apps/web` 及 `packages/types` 目录变更，自动触发 Cloudflare Pages 部署。

* `deploy-api.yml`：监听 `apps/api` 及 `packages/prisma`、`packages/types` 目录变更，自动触发 Cloudflare Workers 部署。

* 两个 workflow 互相独立，任一失败不影响另一服务，支持独立回滚。

**部署优势**

* 前端改动（UI、样式、页面逻辑）不触发后端重新部署，降低部署风险。

* 后端接口变更不影响前端服务可用性。

* Cloudflare Pages 与 Workers 均在同一 Cloudflare 账号下统一管理，域名、CDN、访问控制集中配置。

* 零出口流量费用，全球 CDN 自动分发。

内容访问流程示例：用户提交 MetaMask 签名 → Hono 路由（运行于 Cloudflare Workers）验证签名 → Prisma 查询 Supabase 确认购买记录 → 若验证通过，Worker 为对应 R2 对象生成一个时间受限的预签名 URL 并返回给客户端 → 用户通过该预签名 URL 访问课程内容（URL 为短时效，过期后不可用）。

内容访问流程示例：用户在前端提交 MetaMask 签名 → Cloudflare Worker 接收并使用 Ethers.js 查询链上购买记录并验证签名合法性 → 若验证通过，Worker 为对应 R2 对象生成一个时间受限的预签名 URL 并返回给客户端 → 用户通过该预签名 URL 访问课程内容（URL 为短时效，过期后不可用）。

### 数据存储与隐私

* 课程元数据和简要信息上链公开存储，便于内容查验和生态透明。

* 课程内容（视频、文档等大文件）存储在 Cloudflare R2（示例免费额度：10GB 存储、1000 万次读取/月）。对内容的访问由 Cloudflare Workers 控制：Workers 在验证用户签名并确认链上购买记录后，生成短时效的 R2 预签名 URL 提供给用户，避免公开暴露 R2 对象。

* 关系型数据（用户信息、课程元数据的副本、购买记录、AAVE 质押记录）存储在 Supabase PostgreSQL，并启用 Row Level Security（RLS）作为数据访问控制的额外保护层，防止越权访问。

* 所有用户操作均基于钱包签名，避免用户敏感信息明文存储。实行 nonce 随机数机制，防止签名重放攻击。不对外开放 R2 的直接公共访问；所有对课程内容的访问必须通过 Cloudflare Workers 的验证与授权流程。

### 扩展性与性能

* Cloudflare 免费层容量（早期测试阶段）：Cloudflare Workers 免费层：10 万次请求/天，CPU 时间上限 10ms/次，可支撑约 500～1000 个日活用户。Cloudflare R2 免费层：10GB 存储、1000 万次读取/月，课程内容分发基本够用。Supabase 免费层：提供约 500MB 数据库存储与每月约 50 万次 API 调用，足够早期测试阶段使用；付费层从 $25/月 起，支持更大存储与更高并发连接数，扩展路径清晰。Cloudflare 付费层容量（正式运营阶段）：Workers 付费计划 $5/月：1000 万次请求/月包含，超出按 $0.30/百万次计费，CPU 时间上限提升至 5 分钟/次。付费层可轻松支撑数万日活用户，对于课程学习平台绰绰有余。Cloudflare 全球 300+ 边缘节点自动分发，理论上无并发天花板，费用随流量线性增长，无需手动扩容。真正的性能瓶颈：平台实际并发瓶颈不在 Cloudflare，而在 Sepolia 测试网本身的 TPS 限制——链上交易（购买课程、approve 授权、AAVE 质押）受测试网出块速度影响，用户需等待链上确认。建议前端对所有链上交易做异步状态轮询与友好的等待提示，降低用户因链上延迟产生的流失。对链上合约操作进行最小化 Gas 消耗设计，减少用户成本与等待时间。Cloudflare Workers 侧的 API 响应（签名验证、课程元数据查询）目标响应时间 <200ms，与链上交易解耦，互不阻塞.

### 潜在挑战

* 钱包签名体验与教育，需降低新手误操作概率。

* AAVE 存取失败需完备异常处理，用户资产保护。

* 前后端需对签名风控做全流程自测与合约安全审计。

* Cloudflare Workers 与 R2 的免费层限制需关注：Workers 免费层请求数（示例：100,000 次请求/天）与 R2 读写额度在早期可能满足测试需求，但随使用增长需监控并评估升级以避免服务瓶颈或额外成本。

* 国际化内容维护成本：所有新增功能的用户可见文案需同步维护中英文两份翻译，建议在开发流程中将 i18n 文案更新作为功能完成的必要条件之一。

### 建议阶段划分

第一阶段：项目骨架搭建（第 1 周）

* 关键交付：初始化 pnpm Monorepo 结构，创建 apps/web（Next.js）、apps/api（Hono + Workers）、packages/contracts、packages/prisma、packages/types、packages/i18n 六个工作区；配置 Tailwind CSS 与 shadcn/ui；Supabase 项目初始化，Prisma schema 基础表结构建立，验证 Prisma + Supabase PgBouncer 在 Workers 环境下连通；Cloudflare Pages + Workers 部署流水线跑通。
